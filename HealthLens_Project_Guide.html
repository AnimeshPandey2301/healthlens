🔬 HealthLens — Complete Project Guide

Mentor-Ready Reference | Built with Next.js 16 + Supabase + PostgreSQL | India-focused Health Awareness Platform

1. What is HealthLens?

HealthLens is a free, educational health-awareness web app built for India. It replaces chaotic Google searches with curated, medically-grounded information. It is not a diagnostic tool — it helps users understand symptoms, find nearby doctors, order medicines, and access emergency first-aid guides.

Target Users: General public (especially rural/semi-urban India), patients, students, caregivers.

Problem Solved: Most Indians search random sites when ill and get misinformation. HealthLens provides structured, WHO/AIIMS-verified health awareness that even works on 2G.

2. Features

  • Symptom Checker — Search & select up to 8 symptoms; filter by age, gender, duration; get ranked condition matches with severity (Low / Moderate / High)
  • Doctor Finder (Map) — Uses your GPS + OpenStreetMap/Overpass API to show nearby hospitals, clinics, pharmacies on an embedded Leaflet map with directions
  • Medicine Ordering — Browse 30 medicines across 5 categories; add to cart; verify identity via OTP (Twilio); choose payment method; order saved to DB
  • Emergency First Aid — Step-by-step CPR/first aid guides; quick-dial 112 & 102
  • Diet Plan — Science-backed vegetarian meal plans by age group (toddler → senior)
  • User Dashboard — View symptom session history, severity badges, medical profile
  • Auth (Supabase) — Register/Login with email+password; JWT sessions; protected routes
  • Medical Profile — Store blood group, allergies, chronic conditions
  • PWA Support — Works offline (next-pwa); installable on mobile

4. Architecture (Simple)

Client-Server Model: The browser (client) displays the UI. When you click a button, it calls a Next.js API Route (server). The server talks to PostgreSQL via Prisma and returns data. All API routes run as serverless functions on Vercel/server.

Browser (React) → Next.js API Route → Prisma ORM → PostgreSQL (Supabase)

Auth layer: Every protected API route first calls supabase.auth.getUser() to verify the JWT token from the browser cookie before processing requests.

5. How Data Flows — Symptom Checker

  1. User types "fever" → debounced call to POST /api/symptoms/search
  2. Server queries Prisma → Symptom table → returns matching symptoms
  3. User selects symptoms → Zustand store holds them in memory
  4. User clicks "Analyse" → POST /api/symptoms/analyse called with symptoms + filters
  5. Server matches symptoms against Conditions table → ranks by match score
  6. Results stored in pendingSessionMap (in-memory) + DB via Prisma
  7. User redirected to /results page → reads results from Zustand store

6. How Login Works

  1. User submits email + password on /auth/login
  2. Supabase Auth SDK called: supabase.auth.signInWithPassword()
  3. Supabase returns a JWT token + sets it as an HTTP-only cookie
  4. Every page load: Next.js middleware checks cookie → if missing, redirects to login
  5. Server components call supabase.auth.getUser() to get current user
  6. On logout: supabase.auth.signOut() clears cookie → redirect to home

7. How Doctor Map Works

  1. Page loads → browser requests GPS via navigator.geolocation
  2. Leaflet map initialised on a <div> container; OpenStreetMap tiles loaded
  3. Overpass API queried with GPS coordinates: fetches hospitals, clinics, pharmacies within 3km
  4. Results rendered as colour-coded markers on map
  5. Search bar uses Nominatim geocoding + Overpass for text queries
  6. Specialty chips (Cardiologist, Dentist…) trigger tag-based Overpass queries near user

8. How OTP Ordering Works

  1. User adds medicines to cart (Zustand + localStorage persist)
  2. Enters phone number → POST /api/otp/send → Twilio sends SMS
  3. OTP hash stored in otp_records table (bcrypt hashed)
  4. User enters 6-digit OTP → POST /api/otp/verify → hash compared
  5. On success → user selects payment → POST /api/orders/create
  6. Order saved to medicine_orders table → success screen shown

10. Important Concepts Explained Simply

JWT (JSON Web Token)

A digital "ID card" given after login. It's a small encrypted string stored in a cookie. Every request the browser makes sends this token; the server checks it to confirm "yes, this person is logged in." Supabase handles all JWT creation & validation automatically.

API (Application Programming Interface)

A way for two parts of software to talk. Example: when you click "Analyse Symptoms," the browser sends a message (API call) to the server saying "here are my symptoms." The server processes it and sends back results. In HealthLens, all API routes live under /app/api/.

Async Operations

Things that take time (fetching from DB, calling Twilio, loading map). Instead of freezing the screen, JavaScript uses async/await — it starts the task, shows a loading spinner, and updates the UI when done. Example: the spinning loader while the map fetches doctors.

Prisma ORM

Instead of writing raw SQL like SELECT * FROM symptoms, Prisma lets you write TypeScript: prisma.symptom.findMany(). It's safer (no SQL injection), type-checked, and generates the query automatically. Schema defined in prisma/schema.prisma.

Zustand (State Management)

Imagine a shared notebook that any component can read or write. When you add a medicine to cart in one component, the cart icon in the navbar instantly updates. Zustand holds this shared data in memory. Cart data is also saved to localStorage so it survives page refresh.

Row Level Security (RLS)

A Supabase/PostgreSQL feature. Even if someone has the DB URL, they can ONLY see their own data. Rules like "a user can only read their own symptom_sessions" are enforced at the database level — not just in code. This is in supabase/rls-policies.sql.

11. Viva Questions & Answers

Q1: What is HealthLens and what problem does it solve?

HealthLens is a free health-awareness platform for India that gives structured symptom information based on WHO/AIIMS data, replacing unreliable Google searches. It is NOT a diagnostic tool.

Q2: Why did you choose Next.js?

Next.js supports both Server-Side Rendering (for SEO) and API Routes (backend) in one framework — no need for a separate Express server. The App Router enables server components that fetch data directly.

Q3: What is Supabase and why use it over Firebase?

Supabase is an open-source Firebase alternative built on PostgreSQL (relational). We chose it because it gives us structured data, SQL queries, Row Level Security, and built-in Auth — all in one hosted service.

Q4: How does authentication work?

Supabase Auth issues a JWT token on login stored in an HTTP-only cookie. Next.js middleware checks this cookie on every route. Server components verify the user with supabase.auth.getUser().

Q5: What is Prisma and why use it?

Prisma is a TypeScript ORM. It converts TypeScript method calls into SQL queries. Benefits: type safety, auto-completion, no SQL injection, easy migrations. Schema is defined in prisma/schema.prisma.

Q6: How does the doctor map work without Google Maps?

We use Leaflet (open-source map library) with free OpenStreetMap tiles. Doctor data is fetched from Overpass API (crowdsourced healthcare data from OSM). Zero cost, no API key required.

Q7: What is OTP and how is it secured?

OTP = One-Time Password. Twilio sends a 6-digit SMS. The OTP is hashed using bcrypt before storing in DB. On verify, the entered code is bcrypt-compared with the hash. Expires in 10 minutes.

Q8: What is the role of Zustand in this project?

Zustand manages global client state: the medicine cart and symptom selection. It persists the cart to localStorage so items survive page refresh. It replaces prop-drilling across multiple components.

Q9: How do you validate user inputs?

Zod schemas validate all API request bodies server-side. For example, the profile API uses a Zod schema to ensure blood group is one of 8 valid values. React Hook Form handles client-side form validation.

Q10: What is Row Level Security (RLS)?

RLS is a PostgreSQL feature where database-level rules restrict data access. Even if an attacker bypasses the API, the DB itself only returns rows belonging to the authenticated user's ID.

Q11: Why use Overpass API instead of Google Places API?

Google Places API costs money per request. Overpass API is completely free, provides real healthcare data from OpenStreetMap's global contributor database, and has no usage limits for educational projects.

Q12: What is PWA and did you implement it?

PWA = Progressive Web App. With next-pwa, HealthLens can be "installed" on a phone's home screen and works offline. Emergency first-aid guides load even without internet — critical in India's rural areas.

Q13: How does symptom matching work?

When symptoms are submitted, the server queries the Conditions table. Each condition has a list of associated symptoms. The overlap between user symptoms and condition symptoms is calculated as a match score. Results are ranked highest-score first with severity level.

Q14: How is patient data kept private?

Compliant with India's DPDP Act 2023. RLS ensures users only see their own data. No data sold to third parties. JWT tokens are HTTP-only (not accessible by JavaScript). Sensitive fields like OTPs are bcrypt-hashed.

Q15: What happens if the database is not set up?

The Dashboard gracefully handles DB errors with a try-catch block. If tables don't exist, it shows a friendly "DB setup required" banner with a link to run the SQL setup script — the app doesn't crash.

"HealthLens is a full-stack health awareness web application built for India using Next.js, Supabase, and PostgreSQL. A user can enter their symptoms, and the app matches them against a verified medical dataset to show possible conditions with severity levels. It also has an embedded map powered by OpenStreetMap that finds nearby doctors and hospitals using your GPS location. Users can order medicines from a catalogue of 30 items, verified via SMS OTP through Twilio. The app uses Supabase for authentication with JWT tokens, Prisma ORM for type-safe database access, and Zustand for cart and symptom state management. It works as a Progressive Web App, so it can be installed on phones and partially used offline.
